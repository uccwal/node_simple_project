# 프로젝트 수정:

먼저, 프로젝트의 수정사항을 코드에 적용합니다. 이는 리액트 앱의 소스 코드나 리소스 파일 등을 수정하는 것을 의미합니다.

# 리액트 프로젝트 다시 빌드:

프로젝트를 수정한 후에는 변경된 내용이 반영된 새로운 정적 파일을 생성해야 합니다. 다음 명령을 실행하여 리액트 프로젝트를 다시 빌드합니다.

npm run build

# 도커 이미지 업데이트:

이제 수정된 정적 파일을 포함한 새로운 도커 이미지를 생성합니다. 이때, 이전에 생성한 이미지와는 다른 이름을 사용하는 것이 좋습니다.

docker build -t updated-react-app-nginx .

# 도커 컨테이너 업데이트:

수정된 도커 이미지로 컨테이너를 업데이트합니다. 이는 기존 컨테이너를 중지하고 삭제한 후에 새로운 컨테이너를 실행하는 것입니다.

먼저, 기존 컨테이너를 중지합니다.
docker stop <container_id>

삭제
docker rm <container_id>

새로운 컨테이 실행
docker run -d -p 8080:80 updated-react-app-nginx


# 웹 브라우저에서 확인:

마지막으로, 웹 브라우저에서 http://localhost:8080로 접속하여 수정사항이 반영된 리액트 앱을 확인할 수 있습니다.
